Algorithm and Data Structures Problems



    A&D Problem #1


  What is Computer Science?


  Computer Science is the study of problems, problem solving, and the 
  solutions that come out of the problem-solving process. When faced with a
  problem, the goal of the computer scientist is to develop an algorithm. 
  An algorithm is a step by step list of instructions for solving the problem.

    A&D Problem #2


  What does it mean for a problem to be computable?


  When a problem is computable, it means that there exist an algorithm for
  solving the problem. Problems that are not computable are problems where
  no algorithm exist for solving the problem. Another way to think of
  computer science is the study of problems that are computable and not
  computable.

    A&D Problem #3


  What is abstraction? 


  Abstraction lets you think of the big picture without worrying about the 
  details.

    A&D Problem #4


  What is the difference between an algorithm and a program? 


  An  algorithm  is a step by step list of instructions to solve a problem.
  A  program  is an algorithm that is encoded in a specific programming language 
  so it can be excuted by a computer. There can be many programs using the same 
  algorithm depending on the programer and the programming langugae being used.
  Wihout an algorithm there can be no program.

    A&D Problem #5


  Why study Algorithms? 


  By studying different algorithms we can start to recognize patterns so that
  the next time we run into a similar problem, we are better able to solve it.

    A&D Problem #6


  What is algorithm analysis? 


  Algorithm analysis is concerned with comparing algorithms based upon the 
  amount of computing resources that each algorithm uses. Computer resources 
  include 1) the amount of space or memory an algorithm requires to solve the
  problems as well as 2) the amount of time they require to execute. When
  comparing algorithms we want to select the ones that are most efficient in
  using computing resources.

    A&D Problem #7


  What is Big-O notation 


  Big-O notation is a way to measure how well an algorithm scales as the 
  amount of data involed increases. For example, how well will your algorithm
  perform with a 10 element array versus a 10,000 element array.

    A&D Problem #8


  Give the Big-O Performance for T(n)=n+5. T(n) is the time (or number of steps)
  it takes to complete a problem with size n. Explain your answer. 


   The Big-O Performance is O(n). The reason for this is because as n gets 
   large, 5 becomes less and less significant to the final result so we can 
   just drop it.

    A&D Problem #9


  Give the Big-O Performance for T(n)=4n²+8n+300. Explain your answer. 



   The Big-O Performance is O(n²). The reason for this is as n gets large,
   8n and 200 becomes less and less significant. Hence we are only left with
   the dominant term: 4n². However, we also note at 4 becomes
   insignificant as n gets large so we can also ignore the 4.
